## Умножение матриц

Даны две квадратные матрицы \(A\) и \(B\) размера \(N \times N\). Результирующая матрица \(C = A \cdot B\) вычисляется по формуле:
\[
C_{ij} = \sum_{k=0}^{N-1} A_{ik} \cdot B_{kj}, \quad i,j = 0,\ldots,N-1.
\]

В проекте матрицы хранятся построчно (row‑major). При таком способе хранения элемент матрицы `C` с индексами `i` (строка) и `j` (столбец) обращается как:

```cpp
C[i * N + j] = 0.0f;
for (int k = 0; k < N; ++k)
    C[i * N + j] += A[i * N + k] * B[k * N + j];
```

---

## Реализация на CPU

На CPU используется наивный тройной вложенный цикл (файл `src/cpu_multiplier.cpp`):

```cpp
void multiply_matrix_CPU(const float* A, const float* B, float* C, int N) {
    for (int i = 0; i < N; ++i)
        for (int j = 0; j < N; ++j) {
            float sum = 0.0f;
            for (int k = 0; k < N; ++k)
                sum += A[i * N + k] * B[k * N + j];
            C[i * N + j] = sum;
        }
}
```

Алгоритм полностью последовательный. Его вычислительная сложность \(O(N^3)\); при `N = 2000` это порядка \(8 \cdot 10^9\) операций умножения‑сложения, поэтому время работы быстро растёт с увеличением размера матрицы.

## Реализация на GPU (CUDA)

### Распараллеливание на GPU

На GPU каждый элемент \(C_{ij}\) вычисляется отдельной нитью.  
Нити организованы в двумерные блоки `16 × 16` (256 потоков в блоке).

Используются:

```cpp
dim3 blockSize(16, 16);
dim3 gridSize((N + 15) / 16, (N + 15) / 16);
```

Таким образом, сетка блоков полностью покрывает матрицу.  
Например, при `N = 1000` размер сетки составляет `63 × 63` блоков,  
что даёт 3969 блоков и более миллиона нитей, запускаемых параллельно.

### CUDA‑ядро

```cuda
__global__ void multiply_matrix(const float* A, const float* B, float* C, int N) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;

    if (row < N && col < N) {
        float sum = 0.0f;
        for (int k = 0; k < N; ++k)
            sum += A[row * N + k] * B[k * N + col];
        C[row * N + col] = sum;
    }
}
```

Каждая нить определяет свои координаты `(row, col)` через `blockIdx`, `blockDim` и `threadIdx`.  
Проверка `row < N && col < N` нужна для случаев, когда `N` не кратен размеру блока.

## Проведение эксперимента

В `main.cpp` реализован цикл по размерам матриц от `100` до `2000` с шагом `100`. Для каждого `N` выполняются следующие шаги:

1. Генерируются случайные матрицы `A` и `B` (равномерное распределение на `[0, 1]`).
2. Вычисляется произведение на CPU и измеряется время с помощью  
   `std::chrono::high_resolution_clock`.
3. Вычисляется произведение на GPU: данные копируются на устройство,  
   запускается ядро `multiply_matrix`, результат копируется на хост.
   Время ядра измеряется с помощью CUDA‑событий (`cudaEvent_t`).
4. Результаты CPU и GPU сравниваются поэлементно с допуском `ε = 1e-5`.
5. Размер матрицы, время CPU, время GPU и флаг корректности сохраняются в `results.csv`.

Корректность CPU‑реализации дополнительно проверяется сравнением на малых размерностях: максимальная разница с эталоном составила `1.1920929e-07`, тест корректности CPU пройден.

Для нескольких размеров матриц проверялась максимальная абсолютная ошибка между результатами, вычисленными разными способами:

```text

--------------------------------------------
   100            1.335144e-05        OK
   200            4.577637e-05        OK
   500            9.155273e-05        OK
  1000            2.136230e-04        OK
```

Во всех случаях отклонения укладываются в ожидаемые границы ошибок типа `float`  
и считаются приемлемыми.

## Сборка и запуск

### Требования

- Linux  
- GCC ≥ 7  
- CUDA Toolkit ≥ 11  
- CMake ≥ 3.18

### Команды

```bash
git clone https://github.com/<your-username>/CudaMatrixMultiplication.git
cd CudaMatrixMultiplication

mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j4

./MatrixMultiplication
```

После запуска программа создаёт файл `results.csv` с измерениями времени,  
который далее используется в `lab0.ipynb` для построения графиков и анализа.
